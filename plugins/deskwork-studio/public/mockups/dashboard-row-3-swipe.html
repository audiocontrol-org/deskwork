B · row #01 swiped left · 3 action buttons revealed (Approve · Cancel · Scrap) · contextual primary first (green Approve since In Review) · tap an action OR tap row to dismiss · other rows still at-rest

PatternNo buttons visible at-rest. Swipe left to reveal action drawer. Tap row to open. Pure iOS Mail / Things / Reminders idiom. The visible row is just slug + title + meta — the cleanest possible composition. Swipe left and 2-3 action buttons slide in from the right (filled with stage-action colors: green Approve / blue Iterate / red Cancel / kraft Scrap). The set of buttons is contextual: in-review rows show Approve+Cancel+Scrap; iterating rows show Iterate+Cancel+Scrap; rows with no review state show Cancel+Scrap. Tap an action OR tap the row body OR swipe the row back right → dismiss.

Row height (default state)~70px stable. Same as Direction B. The action drawer adds zero vertical real estate — it's hidden behind the row foreground until summoned. The « hint on the right edge is 14×14 ink — minimal but discoverable.

Trade-offDiscoverability — the operator has to KNOW the swipe gesture exists. The « hint on the right edge of every row is the visual signal but it's small (the iOS Mail equivalent gets away with it because every iPhone owner has been swiping inboxes for a decade). For a NEW operator, the gesture is invisible. Mitigation: a coachmark on first dashboard load, OR keep tap-to-open as the most common path so swipe is for power users. Action buttons are full-screen-width tappable when revealed (72px each ≈ Apple HIG minimum), so once the gesture is discovered the action surface is generous.

When it winsFor an experienced mobile operator: fastest possible path to bulk actions. Swipe-swipe-swipe through 5 rows, approving each in 200ms. For SCAN tasks where the operator wants to see ONLY the slugs/titles/states and not the action chrome: this is the cleanest possible row composition.

When it doesn'tFor a new operator who doesn't know the gesture exists: the dashboard appears to have NO actions, just open-to-review. They'll learn it eventually but there's a discovery cliff. Also requires more JavaScript engineering than Directions A/B (touchstart/touchmove/touchend handlers with rubber-band physics, threshold detection, click-elsewhere-to-dismiss). The gesture also doesn't translate to desktop trackpad/mouse — desktop would need a hover-reveal or a click-to-toggle equivalent (or the existing chip-row could stay desktop-only).
