Mockup · accessibility — row overflow button

The ⋮ is broken; we need a standard, and we need to fix it

Operator caught it on phone: "the three dots affordance that opens the per-item menu is so low contrast, I can barely see it." The current implementation uses color: var(--er-paper-3) against var(--er-paper) — a contrast ratio of 1.21:1, which fails every WCAG criterion that touches an interactive control.

This mockup proposes adding an Accessibility section to DESIGN-STANDARDS.md AND picking a treatment for the ⋮ that meets the proposed standard.

Proposed standard for DESIGN-STANDARDS.md

Accessibility — Contrast

The press-check aesthetic is restrained, but restraint is not invisibility. All affordances must meet WCAG 2.1 AA contrast minimums against their adjacent background.

  • Body text: ≥4.5:1 (WCAG 2.1 SC 1.4.3, AA — normal text).
  • Large text (≥18px, or ≥14px bold): ≥3:1 (WCAG 2.1 SC 1.4.3, AA — large text).
  • Interactive UI components (buttons, icons, controls, focus indicators): ≥3:1 against adjacent colors (WCAG 2.1 SC 1.4.11, Non-Text Contrast). This applies to the ⋮ overflow trigger, drawer chips, menu items, FABs, stage tile chevrons — everything the operator can tap.
  • Ornamental press-check chrome (decorative glyphs, kicker dividers, rule lines): ≥3:1. Decorative ≠ invisible — the chrome is what makes the surface read as press-check, so it has to be legible.

When in doubt, run the foreground / background pair through a contrast checker. Document the ratio in the CSS comment.

The ring gives the ⋮ a clear "tappable region" boundary independent of glyph contrast — the affordance reads as a button, not just three dots floating in space.

Reinforces press-check vocabulary (stipple / press-mark / engraved circle motifs already in the design language).

Glyph stays subdued, so the ⋮ doesn't compete with the row title for attention; the affordance is communicated by the ring shape.

Adds 26px of trailing chrome per row (vs 20-24px for the bare glyph) — minor density cost.

Bigger visual change from current chrome; operators may need a beat to learn what the new shape means.
