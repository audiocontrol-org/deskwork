PatternZero new chrome. The existing Actions sheet (already wired in for Edit + decision verbs) becomes the home for everything you can do with this entry — including reaching its supporting materials. Three sections: Document (Edit), Folio (scrapbook items), Decisions (Approve / Iterate / Reject / Cancel). Section headers use the existing dashed-rule pattern from the Format sheet.

Folio rowsEach item is a one-line row: 24px swatch (format-specific gradient like the chips in Mockup 2) + filename in mono + format chip + size. Reads as a press-check directory listing. Per-format chip color (red PNG / blue MP3 / green MD) carries the same accent map already shipped. Tapping a row opens the file in the standalone scrapbook viewer (or inline lightbox for images).

Section meta"Folio" header carries an open viewer ↗ link in proof-blue — preserves the standalone-viewer affordance without making it a full row.

Sheet height70% (vs. 60% for Notes / Format sheets) because the Actions sheet now carries more content. Drag-handle still pulls to 90% / dismisses to 0%. Sheet body scrolls cleanly between sections.

StrongMost minimal of the three directions. No new tab, no new strip, no new visual artifact. Discoverability is concentrated in the Actions sheet — the operator already opens this for decisions; Folio is right there too. Semantic placement: Folio is "context for the decision," and lives next to the decisions in the same sheet.

WeakDiscoverability cost is the trade-off. The operator has to know that Actions also holds the scrapbook. The Notes count is at-a-glance via the tab badge; Folio count requires opening the sheet. Mitigation: the Actions tab could also surface a small kraft dot indicator when the entry has folio items (similar to a notification dot). Or carry a count beneath the Actions glyph similar to Notes (but two count badges on one tab gets noisy). The header's "5 folio items · 4 decisions" gives the at-a-glance count once you've opened the sheet.
