The canonical source

"You never leave the agent for a static tool. You talk to it. It does the work." The agent's toolkit isn't a static configuration you set up once; it evolves, under your direction, in response to what the work actually needs.

The blog post is the canonical source. Everything below derives from it.

What the thesis means

The agent (Claude, in a Claude Code session) is the primary tool. Every other piece of deskwork — CLI verbs, the studio, doctor rules, sidecars, the calendar — exists to be read, edited, and orchestrated by that agent. Static tools (opaque code the agent can't see or evolve, server endpoints that hide multi-step logic, UI that takes business-logic actions without surfacing what it's doing) are what the thesis rejects.

B · FAB tapped, stack menu open

Notes on the pattern. A single floating action button (60px circle, red-pencil) sits in the bottom-right thumb zone. The article body fills the entire viewport below the strip — no permanent secondary chrome eating space. A small badge on the FAB carries the open-notes count so the operator sees state at a glance even with the menu closed.

Tapping the FAB reveals a vertical stack of action chips above it (Outline, Notes, Approve, Iterate, Reject). Each chip is right-anchored, has a per-action color accent (red for navigation, green for approve, blue for iterate, red for reject), and slides in with a brief stagger. The FAB itself rotates 45° to become a close-button — the spatial metaphor matches the menu's reveal motion.

Tapping a navigation chip (Outline / Notes) opens a modal sheet that occupies the lower 70% of the viewport (similar to Mockup 1's sheet). Tapping a decision chip (Approve / Iterate / Reject) copies the skill command to clipboard and shows a brief confirmation — same THESIS Consequence-2 routing as desktop.
