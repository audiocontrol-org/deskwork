# Deskwork architectural thesis
 
Read this before touching any deskwork code.
 
This file exists because the thesis was implicit in the project's design but not written down anywhere discoverable.
Format · markdown Tap to insert · double-tap holds open

B · format sheet open · press-check keys grouped by function · 56% sheet height

PatternSame bottom-tab-bar idiom as the review surface — 64px, three tabs, glyph + label + count badge, red-pencil active accent. The editor inherits the operator's muscle memory: tabs sit where they're expected, sheets slide up the same way, the article is always visible behind a scrim. The operator transitions Review ↔ Edit and the bottom of the screen barely changes shape.

TabsFormat opens a sheet of markdown press-check keys (H1/H2/H3, B/I/code/link, list/numbered/quote/fence) grouped by function. Notes opens the same sheet from the review surface (4 / 5 / 3 — always the live count). Save is the third slot — green when dirty, faded when clean, with a glowing dot and the kicker turning green to mirror the stamp on top. Tap = direct save.

Mode (Source ↔ Preview)Lives in the strip on the right side, not in the tab bar — because mode is a view of the same thing, not a peer action. Two-state pill in proof-blue. No Split (desktop luxury, hostile on phone per #239).

Stage stamp turns green when editingSubtle but loud — the rotated stamp on top has been the press-check signature throughout. In edit mode it shifts to stamp-green, signaling "you are mutating this entry." Closes the loop on #228 (operator can't tell the editor is dirty) and #241 (chrome should match the writing context, not just be dismissable).

Why this is rightThe review surface's mobile rebuild was operator-validated. This direction reuses the SAME chrome shape with editor-flavored verbs. The cost of remembering the editor's UX is zero — it's the review UX with three different verbs. The trade-off vs. Mockup 1: more chrome at rest (96px top + 64px bottom = 160px), fewer accessory affordances above the keyboard. The win is continuity; the cost is some viewport real estate.
