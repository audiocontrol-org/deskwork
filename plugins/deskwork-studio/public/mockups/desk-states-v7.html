§ 2 Anatomy · stage tile (3 states)

Three visual states per stage tile: at rest, expanded, empty. Same composition; visual treatment varies to communicate state.

State Background Glyph + name Chevron Tappable? Notes
At rest --er-paper full --er-ink 0° (›) yes Default. Hover lifts to --er-paper-2.
Expanded --er-paper-2 + 2px red-pencil top inset rule name shifts to --er-red-pencil (or --er-proof-blue for shortform) 90° (↓) yes — tap closes Rows render inline below the tile in a row-group container.
Empty (count = 0) --er-paper @ 0.55 opacity name in --er-faded hidden no (cursor: default) Pipeline shape still visible. "This stage exists; nothing here right now."

§ 3 Interaction spec

Single-expand · within section Within the longform pipeline section, only one stage may be expanded at a time. Tapping a second stage tile closes the previously-open one and opens the new one. Within the shortform section, the same rule applies — one platform expanded at a time.
Independent · across sections The two sections operate independently. The operator may have one longform stage AND one shortform platform expanded at the same time — useful when cross-referencing pipeline state with social distribution. The "Adjacent tools" section's tiles are non-expandable (they navigate to leaf surfaces in Phase 3+).
Row interactions (inherited from v0.20) Each row's trailing ⋮ opens a popover menu of stage-aware verbs (the same popover idiom as v0.20 row affordance and v7 masthead menu). Swipe-left on a row reveals an action drawer (also v0.20). Tap the row body navigates to the entry-review or shortform-review leaf surface.
Empty-state semantics Per DESIGN-STANDARDS.md § Principles · Favor structure over scrolling: empty hierarchy nodes still have a place in the structured view. Showing "Ideas: 0" or "Instagram: 0" is a structural signal — the absence of items is information about the pipeline shape. Empty tiles render dimmed (opacity 0.55), with the chevron hidden and tap target disabled.

§ 4 What this Desk commits to

The full IA, written down
  • Longform pipeline section · 8 stage tiles, in flow order (Ideas → Cancelled), with single-expand behavior.
  • Shortform · by platform section · 4 platform tiles (LinkedIn · Reddit · YouTube · Instagram), with same single-expand behavior. Each platform shows workflow count; tapping reveals workflow rows.
  • Adjacent tools section · 2 future tiles (Folio · Files) reserved for Phase 3. Dimmed with phase-tag, non-expandable until those surfaces ship.
  • Glyph mapping per DESKWORK-STATE-MACHINE.md: ◇ § ⊹ ✎ ※ ✓ ⊘ ✗. Platform tiles use color-coded badges (in red · r/ blue · @ + IG kraft).
What this DOES NOT decide
  • The row content design (slug + title + date + ⋮) is inherited from v0.20; if we want to refine row composition further, that's a separate pass.
  • Filter / sort affordances WITHIN a section — operator may eventually want to filter Drafting rows by stale-age, etc. Not in this Desk's scope; would live as a section-level mini-bar above the row-group.
  • Stage-tile reorder or rearrangement — locked to flow order per the state machine. Future operator-customizable orderings would need separate spec.
  • Shortform-by-platform sub-grouping — could eventually surface "approved · iterating · scheduled" within each platform, but for v0.21 the platform-with-workflow-rows is the shape.
How Desk evolves in Phase 3+
  • Phase 3 ships the Folio + Files surfaces; the dimmed tiles become live (lose the phase-tag, become tappable, navigate to /dev/scrapbook + /dev/content).
  • If more cross-cutting destinations arrive, they slot in as additional tiles in the Adjacent tools section. No bar geometry changes.
  • Section ordering may evolve — pipeline always first (the main work), then social distribution, then adjacent tools. New sections are appended.
