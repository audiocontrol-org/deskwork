§ 2 Fix · popover anchored under ⋮ (not slide-up sheet)

v6 used the Phase 2.1 slide-up sheet for the masthead menu. That violates the user's spatial model: trigger at the top, reveal from the bottom. v7 uses a dropdown popover anchored UNDER the ⋮ with an up-pointing arrow registered to the glyph's x-position. Same idiom as the v0.20 dashboard-row ⋮ — fully consistent vocabulary.

The Apple HIG pattern we're matching iOS / macOS: any nav-bar "More options" affordance opens a popover with an arrow pointing at the trigger. Examples: Mail's More menu, Safari's tab actions, every native iOS app's nav-bar overflow. The popover emerges from the trigger spatially. Slide-up sheets are for bottom-toolbar affordances or modal-style content. The two patterns are not interchangeable.

Phase 2.1's createSlideUpSheet stays in use for bar-triggered sheets (Actions / Outline / Notes / Format / Compose). The masthead menu doesn't use it — it uses a positioned-absolute popover with the same dismiss patterns (scrim tap, ESC, trigger tap-again).

§ 4 What v7 settles

v7 specs (revisions to v5/v6 standards draft)
  • Masthead alignment: align-items: center on the masthead grid + align-self: center on side glyphs and center stack. No element overlaps any border rule.
  • Masthead height: 102px on mobile (was 82px), giving content area more breathing room.
  • Masthead bottom padding: 14px clearance between content and border-bottom.
  • Menu reveal: dropdown popover anchored UNDER the ⋮, with up-arrow registered to glyph's x-coord. Position: top: 92px; right: 0.7rem; width: 270px; The popover does NOT use Phase 2.1's createSlideUpSheet — it's a simple positioned-absolute element with scrim + dismiss handlers.
  • ⋮ vocabulary unified: Both the v0.20 row ⋮ and the v7 masthead ⋮ use the same glyph AND the same reveal pattern (popover, not sheet). One vocabulary across the studio.

§ 5 Standards update · revised from v5+v6

The "§ Studio navigation model" section gets three more edits
  • Glyph alignment: "Both the back and menu glyphs vertically center with the masthead's content stack. The masthead's bottom-border rule, where present, is drawn with clearance below all content — no element overlaps any rule."
  • Menu reveal pattern: "The masthead's ⋮ opens a dropdown popover anchored under the glyph, with an up-pointing arrow registered to the glyph's x-position. The popover is dismissed by scrim tap, ⋮ tap-again, or Escape. The masthead menu does NOT use Phase 2.1's createSlideUpSheet — that primitive is reserved for bottom-anchored affordances (bar-triggered sheets, Compose chip)."
  • ⋮ vocabulary: "⋮ is the studio's universal overflow affordance. It appears in two scopes: dashboard-row level (row options popover, v0.20) and masthead level (studio menu popover, v7). Both use the same glyph AND the same reveal pattern. Operators parse the scope by location."
