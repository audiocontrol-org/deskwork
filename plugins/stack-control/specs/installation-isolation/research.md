## Decisions

- **R1 — One resolver, one record.** Every state-writing verb resolves a single typed `Installation` (root + config) via the existing 009 primitives (`resolveInstallation` / `resolveCodebaseBoundary`) exactly once at entry, and threads it down. No downstream function re-derives placement from a path string. Rationale: the split brain (#3 vs #4) exists because resolution is re-done locally; one threaded record closes the class. Alternatives: per-module resolution helpers (rejected — re-creates drift).
- **R2 — Retired flag set (Clarification 2026-06-10).** `--repo-root` is REMOVED from the state-writing verbs: `govern`, `audit-barrage`, `audit-barrage-lift`, `slush-findings`, `scope-widen`, `scope-inventory` (and the barrage config loader's param chain). Explicit anchoring is only `--at <dir>` (resolves the installation enclosing `<dir>`; already the convention on install-scope-discovery/scope-export). Read-only verbs that take explicit artifact paths (`spec-check`, `execute-check`, `scope-export --manifest`) are unchanged — the retirement is about placement, not about naming a file to read. Old `--repo-root` invocations hit the existing unknown-flag usage error (exit 2) — loud per the operator decision.
- **R3 — Git derived at the installation.** The payload assembler invokes `git -C <installation.root>` with `--relative` (diff) and cwd-relative `ls-files`; the git toplevel is derived internally (`git rev-parse --show-toplevel`) ONLY to (a) detect the legacy half-installation at the toplevel (R6) and (b) anchor the cross-tree feature fold: when the resolved feature root lies outside the installation subtree, the payload adds an explicit, labeled second diff arm scoped to the feature root (pathspec `-- <featureRoot>`), so spec artifacts are never silently absent (US3/FR-005). Alternatives: relocate-first and skip the fold (rejected — leaves the transitional layout unauditable until US6 lands).
- **R4 — Feature anchor is designated, not default.** Writes to the resolved feature root (audit-log, lift, evidence, manifests) remain legitimate wherever that root lives, but when it lies OUTSIDE the installation the verb announces the cross-tree anchor once on stderr. The isolation probe exempts exactly: OS tmpdirs, the resolved feature root, and announced explicit overrides (FR-008's list). Rationale: the feature root is operator-visible state with its own resolver; silently re-homing it would break the append-only ledgers.
- **R5 — Probe mechanics.** A vitest harness builds a nested fixture (outer git repo → inner installation via `stackctl setup`-equivalent seed), snapshots the outer tree (recursive path+mtime+size listing excluding the installation subtree), runs each state-writing verb (table-driven; the verb list IS the R2 set plus backlog capture/import), and asserts a byte-identical snapshot after. Runs as a normal suite member (SC-001/SC-002); local-only smokes stay out of CI per project rules — the probe is plain vitest, so it rides the existing `npm test` path.
- **R6 — Legacy half-installation detection at the decision site.** When the resolved installation's root differs from the git toplevel AND the toplevel carries `.stack-control/` WITHOUT `config.yaml`, the resolving verb emits a three-part notice (what was found, what is actually read/written, safe migration advice) — the 014-US2 pattern, including its AUDIT-09/-15 lesson: the advice never targets existing operator-tuned files destructively. Fires once per invocation; never fires when no legacy state exists.
- **R7 — Relocation mechanics (US6, this repo).** `git mv .specify specs plugins/stack-control/` (history-preserving); Spec Kit re-roots automatically via its nearest-`.specify` walk-up (verified upstream behavior, TASK-45 research); `feature.json`'s relative `feature_directory` stays valid against the new root. Consumers updated in the same change: the repo-root `CLAUDE.md` SPECKIT pointer, `resolveFeatureRoot`'s `<repoRoot>/specs` lookup (becomes `<installation>/specs` first, grandfathered legacy lookups retained per descriptive-naming's forward-only decision), and the extension wiring paths under `.specify/extensions*`. Numbered spec dirs move as-is (grandfathered names). The relocation is the LAST story; everything before it must pass with the cross-tree fold.
- **R8 — Test placement.** Verb-level tests under `src/__tests__/` per the 014/R9 precedent; the isolation probe lives there too (it is a verb-level contract). Fixtures via mkdtemp on real fs, never fs mocks.
