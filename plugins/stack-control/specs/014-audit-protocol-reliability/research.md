## Decisions

- **R1 — US1 reporting + floor mechanics.** Decision: extend the existing summary/exit seam (`renderSummaryLine` + `deriveBarrageExitCode`) rather than adding a new reporting layer; "degraded" = configured model with `stdoutBytes === 0` (timeout or not — the spec's edge case: partial output before a timeout is NOT zero-output degradation, and the warning must not misclassify it). The floor option counts **emitting models** (stdoutBytes > 0) against `min(requested, configured fleet size)` so a one-model fleet doesn't make strict mode unsatisfiable nonsense — a configured-fleet floor shortfall is named as such. Govern passes floor 2 by default (Clarification 2026-06-11), flag-overridable both directions. Rationale: smallest additive change agreeing with the run JSON ground truth. Alternatives: post-hoc run-JSON analyzer verb (rejected — moves loudness away from the moment of failure).
- **R2 — US2 placement.** Decision: detection lives in `loadAuditBarrageConfig` only (the moment the wrong config would silently win), warning on stderr in all legacy-present combinations; no separate doctor rule in this feature (capture-separately if wanted later). Rationale: one detection point at the decision site beats two drifting ones. Alternatives: install-scope-discovery-time migration offer (rejected here — migration writes are a bigger surface; the spec promise is loudness, not auto-migration).
- **R3 — US3 clustering contract.** Decision: union requires `headingsAgree()` (the mechanism proxy); `surfacesAgree()` alone no longer unions and instead only *annotates* same-surface adjacency. `mergeCluster` is unchanged for true merges (cross-model annotation preserved). Rationale: the recorded collapses were all surface-only unions; heading overlap was the discriminator that would have kept the five mechanisms separate. Alternatives: LLM-based mechanism comparison (rejected — nondeterministic merge keys violate the determinism the audit-log depends on); body-similarity scoring (rejected — same nondeterminism risk, more code).
- **R4 — US4 single source of truth.** Decision: `slushRemaining`'s flips carry their status-line locations (the dampener already walked the text); apply consumes exactly that set — `findFindingsByStatus` leaves the apply path. A flip whose recorded location no longer matches at apply time (file changed between dry-run and apply) fails the verb loudly naming the finding. Rationale: spec scenario 2 demands migrate-or-fail-loud, never silent shortfall. Alternatives: reconciling the two walks' keying (rejected — keeps two sources of truth; the divergence class survives).
- **R5 — US5 exclusion mechanism.** Decision: exclude the resolved feature root's `audit-log.md` (and the governance bookkeeping surface under the feature root) from BOTH arms — git pathspec exclusion (`:(exclude)...`) on the committed diff, skip-list on the untracked walk; scope the untracked fold to paths under the feature under audit (resolved via the 013 layout-aware root). Rationale: research confirmed the committed-diff arm also carries the audit-log (lift commits in-range) — fold-only exclusion would not extinguish the AUDIT-28/42/48 generator. Alternatives: prompt-level "ignore the audit-log" instruction (rejected — instruction over mechanism, the exact failure class the thesis names).
- **R6 — US6 seed mechanics.** Decision: on baseline ENOENT inside a widen, invoke the existing install-scope-discovery seeding primitive for the missing state (announced on stderr), then continue the widen; the seeded baseline is the legitimate empty baseline, so clone-derived widening over it is a true "no registered clones" result, not a fabricated default (no-fallbacks rule satisfied — the seed is announced state creation, not silent substitution). Post-seed unsatisfiable clone requests keep the existing loud remediation. Rationale: Clarification 2026-06-11 picked auto-seed; reusing the install primitive avoids a second seeding code path. Alternatives: announced-skip (declined by operator); inline minimal-seed (rejected — duplicates install-scope-discovery).
- **R7 — US7 probe refinement (spec SC-007 measurement).** Decision: the acceptance probe targets path **construction**: grep hits are violations only when they build a filesystem path; fail-loud error-message strings and comments that *name* both layouts (mandated by 013's loudness contract) are compliant and expected to remain. The probe implementation: grep for `001-IN-PROGRESS` excluding `feature-root.ts` + tests, then assert every remaining hit is inside a string literal used for an error message/comment — operationally, the six construction sites convert and the message/comment hits stay. Rationale: research showed the literal grep-empty form would condemn the very loudness 013 added; the spec's intent (no consumer constructs the legacy path) is preserved. This refinement is recorded here and mirrored as a one-line amendment to SC-007.
- **R8 — US8 fault isolation split.** Decision (Clarification 2026-06-11): `list` warns per bad file on stderr and continues; `exists` and both imports' idempotency checks throw `BacklogError` naming the file (existing exit-2 mapping) because a skipped file could hide the ref an idempotency check needs (duplicate-creation risk). All-files-malformed edge: list reports zero healthy items plus the warnings (distinguishable from a clean empty store by the warnings); imports fail loud. Rationale: read paths favor availability; write-adjacent paths favor safety.
- **R9 — Test placement.** Decision: verb-level RED tests under `src/__tests__/` (matching barrage/slush/backlog test placement), governance-surface tests under `tests/` only when they exercise the spec-governance extension — same split vitest.config.ts:8 already collects (013 deviation lesson: put tests where vitest runs them).
