# Session Analytics

## Per-Session Metrics

Each `DEVELOPMENT-NOTES.md` entry should include:

- Messages
- Commits
- Corrections
- Files changed

## Correction Tags

- `[PROCESS]`
- `[UX]`
- `[COMPLEXITY]`
- `[FABRICATION]`
- `[DOCUMENTATION]`

## Pattern Tracking

Look for:

- recurring correction categories
- whether corrections are decreasing
- what zero-correction sessions have in common
- correlation between session shape and correction count
