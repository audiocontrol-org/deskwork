# Session Start

1. Identify the feature from the worktree/branch.
2. Read the feature `README.md` and `workplan.md`.
3. Read the latest `DEVELOPMENT-NOTES.md` entry.
4. Check open GitHub issues.
5. Report:
   - feature and phase
   - recent accomplishments and failures
   - unresolved issues
   - proposed goal for this session
6. Wait for confirmation before coding when the session is a planning/resume flow.
