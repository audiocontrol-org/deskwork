Phase 1 Task 1.3 — pin a comment to a DOM region inside a rendered HTML mockup. Text-range pins flow through @recogito/text-annotator's W3C TextQuote/TextPosition selectors; non-text regions flow through a thin hand-rolled DOM-selector layer that emits a CssSelector with TextQuote + FragmentSelector (pixel-offset) fallbacks.

Text-range: select text inside the iframe; the library emits a TextQuoteSelector + TextPositionSelector. DOM region: click any element (including non-text icons / images / decorative divs); the spike's thin layer emits a CssSelector + TextQuote fallback (if any text content is present) + FragmentSelector pixel offsets.

Note: the text-annotator instance for the iframe is created inside the iframe's own document context (script injection) because @recogito/text-annotator attaches its selection listeners to document, not to the container's ownerDocument. See findings doc § "Anchor model — measured."
