# Phase 1 Data Model: Migrate scope-discovery into stack-control

Entities are the on-disk artifacts + in-memory records the migrated surface operates on. Validation rules trace to FRs; the schemas are ported JSON Schema (R2). All reads/writes fail loud on malformed input (FR-035, Principle V).

## CodebaseBoundary (in-memory)

The resolved single-codebase scope a clone run operates within.

| Field | Type | Notes |
|---|---|---|
| `installationRoot` | absolute path | Nearest-enclosing `.stack-control` config dir (009 `resolve-paths` walk-up). |
| `excludedChildren` | absolute path[] | Nested child-installation subtrees excluded from the scan (009 FR-021). |
| `explicitOverride` | absolute path \| null | Set only when `--root` supplied; otherwise null (default = `installationRoot`). |

**Validation**: resolution MUST fail loud if the walk-up reaches the repo/filesystem boundary with no config (FR-007; never `process.cwd()` fallback). `excludedChildren` are derived, never authored.

## CloneGroup (in `clones.yaml`)

A set of duplicated spans with a disposition.

| Field | Type | Notes |
|---|---|---|
| `id` | string (content-hashed) | Stable across refresh (`clones-yaml.id`). |
| `members` | `{file, startLine, endLine}[]` | The duplicated spans (codebase-relative paths). |
| `lines` | integer | Span length. |
| `disposition` | enum \| absent | `refactor` \| `keep-with-reason` \| `ignore-with-justification`; absent = un-dispositioned. |
| `reason` | string | Required when `disposition` present. |
| Step 0a (refactor only) | `canonicalSide`, `canonicalReason` | Required for `disposition: refactor` (FR-011). |
| Step 0b (refactor only) | `tests`, `testsProof.sha`, `testsProof.demonstration` | Required for `disposition: refactor` (FR-011). |
| `newShapeSummary` | string | Optional (Step 0a branch iii — name-the-new-shape). |

**Validation**: `refactor` disposition is REFUSED unless all Step 0a + Step 0b fields present (FR-011). Schema-violating entries → doctor `clones-yaml-schema-violation`; refactor-incomplete → `clones-yaml-refactor-incomplete`.

**State transitions**: un-dispositioned → (`dispose-clone`/`batch-dispose`) → dispositioned. A NEW group (detector output not in baseline) → surfaced by gate-mode (FR-012); refresh carries dispositions forward across renames (FR-013). A non-pending→pending silent regression is caught by `check-disposition-survivor`.

## CloneBaseline (`clones.yaml` document)

The committed, dispositioned set a codebase is measured against.

| Field | Type | Notes |
|---|---|---|
| `schemaVersion` | integer | Doctor flags stale via `scope-discovery-schema-stale`. |
| `groups` | CloneGroup[] | The baseline set. |
| `boundary` | path | Records which codebase root produced the baseline (per-codebase, R1). |

**Validation**: NEW-detection compares fresh detector output to `groups`; gate-mode exits non-zero on any NEW (FR-012). Malformed baseline → fail loud (FR-035), never treated as empty.

## ScopeManifest (per-feature, generated by scope-inventory)

| Field | Type | Notes |
|---|---|---|
| `feature` | string | Feature slug. |
| `surfaces` | `{path, agent, provenance}[]` | Discovered surfaces with which agent found them. |
| `regimeHoldouts` | section \| absent | Present only when config-activated agents ran (FR-019). |
| `candidates` | `{kind, ...}[]` | novel-shape / discovered-candidate signals (FR-017 — NOT all-clear). |
| `runId`, `stamp` | string | Evidence-trail identity (FR-015). |

**Validation**: schema-valid against `scope-manifest.schema.json` (ported). A green run with non-empty `candidates` MUST NOT be reported all-clear (FR-017).

## DispatchReturn (in-memory, validated by the wrapper)

| Field | Type | Notes |
|---|---|---|
| `searched` | count/list | |
| `included` | list | |
| `excluded` | `{match, reason}[]` | Required enumeration when `searched > included` (FR-024). |

**Validation**: reject on missing grammar labels; reject when `searched > included` and `excluded` doesn't enumerate omissions; reject any `excluded.reason` containing a forbidden-deferral phrase (project-overridable list, `dispatch-wrapper-overrides`). Refactor-marked tasks get the preconditions prelude appended (FR-025). The validator harness's gutted-stub self-check MUST fail if the gate logic is gutted (FR-026).

## ScopeDiscoveryConfig (`.stack-control/scope-discovery/config.yaml`)

| Field | Type | Notes |
|---|---|---|
| `schemaVersion` | integer | Independent of the 009 installation config version (R5). |
| per-agent activation | flags | Which config-activated agents are on. |
| tunables | object | e.g. ast-grep pattern list location, forbidden-phrase overrides, prd stopwords/top-N. |

**Validation**: structural, fail-loud, unknown-key rejection (mirrors stack-control's existing config-loader idiom). Created lazily-and-announced (R5 / 009 FR-016); doctor `scope-discovery-config-missing` surfaces an install hint when absent and a verb needs it.

## Relationships

- `CodebaseBoundary` scopes the clone detector → produces/refreshes a `CloneBaseline` of `CloneGroup`s.
- `scope-inventory` fans discovery agents over the boundary → `ScopeManifest`; config-activated agents read the registries to add `regimeHoldouts`.
- The registries feed the `check-*` verbs and the config-activated agents.
- `ScopeDiscoveryConfig` records activation + tunables + (via 009 resolve-paths) where the registries/baseline live.
