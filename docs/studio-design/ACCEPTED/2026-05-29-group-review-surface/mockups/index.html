Group review surface

Three directions for the group review surface (Tasks 7.3 + 7.4 folded) and three directions for the "Member of:" affordance that appears on member entries' rows in the lane dashboard. Read brief.md for the full design context, constraints, and decision criteria.

Group review surface

3 directions
Direction A
Members-as-list with sticky lane-stage summary strip
Default render = flat member list. Task 7.4's multi-lane composition is reframed as a one-line sticky summary strip at the top — same pipeline-shape signal, no horizontal scroll on mobile.
Trade-off Most operator-readable on narrow viewports; least faithful to the Phase 5 swimlane shape; cheapest implementation.
Direction B
Composed multi-lane default with list toggle
Default render = miniature multi-lane swimlane scoped to the group's members. Section-head toggle flips to flat list. Includes empty-members fallback frame.
Trade-off Honors "structure over scrolling"; reuses Phase 5 swimlane primitive; requires per-group toggle state.
Direction C
Stacked composed-then-list (no toggle)
Both views always visible — compact composed strip on top, flat member list below. No mode state for the operator to track.
Trade-off No toggle to learn; double chrome cost; composed strip drops off-screen as list grows.

"Member of:" badge on lane dashboard rows

3 directions
Direction 1
Pull-tab on row edge
Vertical kraft-color tab on the row's left edge with a member-count badge. Tap expands inline to list parent groups with back-links. Mirrors the marginalia-tab / outline-tab affordance precedent.
Trade-off Most discoverable structurally; introduces a new row-chrome class on the dashboard.
Direction 2
Kicker line above slug
"MEMBER OF v0.18-row-rebuild ↪" line above the row's title in mono caps + kraft color. Multi-parent uses `+N` overflow that opens a popover.
Trade-off Cheap chrome; reads naturally; adds one extra line of vertical space per member row.
Direction 3
Trailing inline chip near ⋮
Small bordered chip immediately left of the row's overflow `⋮` — glyph plus parent count. Tap opens popover.
Trade-off Co-located with existing row affordances; risks clustering with v0.20 overflow vocabulary; the glyph isn't pre-loaded operator vocabulary.
How to pick
  1. Open each mockup at iPhone-13 viewport (390×844) and a desktop viewport (≥1280px).
  2. Pick one direction from each section (A / B / C and 1 / 2 / 3).
  3. Move this proposal from PROPOSED/ to ACCEPTED/2026-05-29-group-review-surface/ with the picks noted in brief.md.
  4. File REJECTED/2026-05-29-group-review-surface-alternatives/brief.md referencing the un-picked directions.
  5. Update DESIGN-STANDARDS.md's change log with the decision.
