Stamp treatment · before / after

Original (desktop reading)
In review Iterating Approved
Rotated −1.2deg, single border. Reads as "rubber stamp pressed at an angle" on desktop.
Revised (mobile reading)
In review Iterating Approved
Flat, doubled border (2px outer + 1px inset paper), wider tracking, ink-bleed text-shadow. Reads as "letterpress tag" — pressed, not tilted.

What changedTwo revisions to the picked Mockup 1. First: tab 3 is now Compose instead of Folio (the soft spot in the original narrative). Compose holds three creation verbs — /deskwork:add, /deskwork:ingest, /deskwork:shortform-start — surfaced as cards that copy the slash command to the clipboard, per THESIS Consequence 2 (UI routes intent; agent does the work). Second: every stamp on mobile is flat (no rotation). The press-check vocabulary stays — cream paper, JetBrains Mono kickers, Newsreader italic display, red-pencil accents — but the EXPRESSION of "this is a stamped mark" shifts from "rubber-stamp-on-paper" (rotation, ink-pressure asymmetry) to "letterpress-tag" (doubled border, wider tracking, ink-bleed text-shadow).

Why rotation read messy on mobileOn desktop, rotated stamps cluster across horizontal real estate — they look like an editor pressed each one independently, with hand variability. The eye reads it as human craft. On mobile, with rows stacked vertically and stamps clustering near each other in tight 22-line-height cells, the same 1.2deg rotation reads as the page is crooked — the angular delta is more visible relative to the row height, and multiple rotations stacked vertically compound. The eye reads it as misalignment, not craft.

How "stamped" still reads without rotationThree signals do the work that rotation used to do on desktop: (1) doubled border — 2px outer ink + 1px inset paper layer, simulating the imprint outline of a rubber stamp on textured paper; (2) wider tracking + small caps — 0.12em letter-spacing reads as engraved/pressed type, not casual italic; (3) ink-bleed text-shadow — 0.4px halo simulates ink seeping into paper fibers, NOT a drop shadow (drop shadows look digital).

Hero strip markSame treatment. The "Press-Check" wordmark on the hero strip used to rotate −1.5deg; revised it sits flat with the same doubled-border, ink-bleed treatment. The 1px red-pencil rule under the strip continues to read as the masthead baseline, so the press-check signal is still there — it's just expressed as letterpress nameplate rather than hand-pressed stamp.

Compose tab — why a third tab and not a FABA floating action button would have been the standard mobile shape, but it breaks the press-check vocabulary (round buttons, Material shadows, Roboto labels) — every other piece of mobile chrome on this surface is a flat letterpress chip. A third tab keeps the chrome consistent: same 64px tabbar, same italic-display glyph + mono label, same red-pencil active accent. The "+" glyph is JetBrains Mono (not italic display) to read as creation operator rather than stage marker — the only typographic deviation in the bar, signaling "this tab does something different."

Compose sheet shapeReuses the established sheet (60% height, drag handle, red baseline, scrim). Each verb is a card with: italic display name, JetBrains Mono command chip in proof-blue (matches Source/Preview accents), prose description in display, and a tap → clipboard · paste in Claude Code footer to make the affordance unambiguous. Order: add (most frequent) → ingest (adoption / one-shot) → shortform-start (advanced).

Implementation noteThe CSS deltas vs. the entry-review surface are minimal — the entry-review's stamp class can keep its rotation in place (single stamp per screen, proud signature element), while the dashboard introduces a .stamp override that drops rotation, doubles the border, and widens tracking. Both stamps inherit the same color palette and font-family. Recommended approach for Task 1.2: add a base --stamp-rotation custom property defaulting to −1.2deg, override to 0 on dashboard rows, and let the rest of the CSS (border, tracking, shadow) cascade.
