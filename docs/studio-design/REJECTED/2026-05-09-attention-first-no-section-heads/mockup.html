B · "All" filter · stage section heads return · within each section, attention-ordered (review-stamped rows first) · empty stages omitted (no Ideas/Outlining/Blocked/Cancelled rendering)

PatternFilter chips become the primary navigation; stage organization becomes secondary. Default chip is ★ Active — the filter shows ONLY rows with a live reviewState (in-review / iterating / approved) regardless of stage. The first thing the operator sees is the small set of entries that need a decision NOW. Each row carries its stage in the kicker (✎ Drafting · 5m ago) so the operator never loses stage identity even when the section head is gone. When the operator picks All, section heads return and rows are sorted attention-first within each stage. Empty stages never render their section head.

Scroll cost (default state)~96px to first attention card. The view-head ("Active · 4 entries need a decision · attention-ordered") replaces multiple stage heads. With 4 attention cards × ~140px each = ~560px, the entire Active list fits in the viewport. Compare current implementation: 746px to first content row. With "All" filter active, scroll to first row is again ~96px (Planned head + first row, no preamble).

Trade-offThe pipeline shape is no longer the FIRST thing the operator sees. They see "Active 4" and a list of 4 cards. To see the full pipeline organization (which entries are in Planned vs Drafting vs Final), they tap "All". This re-frames the dashboard from "calendar of stages" to "queue of attention." For an operator whose mental model IS the calendar, this is a subtle violation. For an operator whose mental model is "what do I need to do next," it's a direct match. Stage chips remain available; muscle memory of "tap Drafting → see Drafting" still works.

When it winsOperator opens the dashboard, sees 4 attention cards, taps the most pressing → review surface, makes a decision, returns. Repeat. The dashboard becomes a triage queue rather than a calendar. The empty stages problem disappears because empty stages don't render at all. Compact-density-grade scroll cost without the compact toggle complexity.

When it doesn'tThe operator wants to verify the FULL pipeline shape ("did anything land in Outlining recently?"). The Active filter hides Outlining entirely if it's empty; even the All filter omits empty stages. The operator can no longer see "0 in Outlining" at the dashboard level — they'd have to know that Outlining is missing from the rendered list and tap a hypothetical "Show empty stages" affordance (not in this mockup). Compactness has a cost: the structural awareness of the pipeline diminishes.
