## Method

The script walks the project's `.deskwork/entries/` sidecars and `.deskwork/review-journal/pipeline/` workflow records. For every applied longform / outline workflow that maps to a sidecar (`state === 'applied'`, `contentKind ∈ {longform, outline}`, `entryId` matches a sidecar), the script:

1. Loads the workflow's `currentVersion` snapshot from the per-version history-journal event at `.deskwork/review-journal/history/<timestamp>-version-<workflowId>-v<n>.json`.
2. Loads the entry's current on-disk content from `sidecar.artifactPath`.
3. Computes a content diff and classifies as: `identical` / `whitespace-only` / `frontmatter-only` / `non-trivial` / `approved-snapshot-missing` / `current-content-missing`.

Only `non-trivial` diffs warrant re-review under the new entry-keyed surface. `whitespace-only` and `frontmatter-only` diffs are noise; `identical` is no drift.

## Provenance / changes from v1

This is the v2 audit doc. v1 (committed in `1028914`) used an iteration-count-only signal and picked the comparison workflow record by `readdirSync` order, which is non-deterministic. The Phase 34e implementation audit (`2026-05-03-phase-34e-implementation-audit.md`) flagged both findings as high-severity. This v2:

- Filters workflow records to `state === 'applied'` + `contentKind ∈ {longform, outline}` (per the audit's F1 fix).
- Iterates ALL applicable records per entry (not just one), surfacing 4 PRD review cycles instead of conflating them.
- Computes actual content diffs (per F2) — pulls the workflow's version snapshot from the history journal, compares against current on-disk content, classifies as identical / whitespace-only / frontmatter-only / non-trivial / incomplete.
- Per-pair disposition replaces v1's narrative claim that "no diffs were surfaced" (v1 didn't compute diffs).

Net change in conclusion: same outcome (zero entries need re-review), but now substantiated by content comparison rather than asserted via narrative.

The audit script (`scripts/audit-post-pivot-iterations.ts`) is committed alongside this doc and re-runnable.
